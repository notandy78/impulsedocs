Installing impulse

This guide will show you how to install impulse, on a server. Before we start, it should be understood impulse is made to be developed in a locally hosted peer-to-peer server, and it is made to be ran in production on a game server. To learn how to setup impulse, for development on your local machine, take a look at the dev setup guide.

Warning

impulse is a framework, a framework is similar to a game engine, if there is an issue with your schema, it is not the frameworks fault. Please consult the developer of the schema for help regarding those issues.

Installation

1. Install mysqloo onto your server.
2. Make sure you remove ULX if you have it already, it is not compatible with impulse
3. Install FPP onto your server.
4. Turn off MySQL strict mode on your MySQL database.
5. Download the impulse framework and put it in your gamemodes folder.
6. Download your schema of choice and also put it in your gamemodes folder. If you don't have a schema, you can use the skeleton schema.
7. (not required, but reccomended) Install and setup GExtension.
8. Goto garrysmod/data on your server and create a folder called 'impulse'.
9. Inside the impulse folder create a file called config.yml and paste in the config below:

 db:
  ip: "mysql server ip here"
  username: "db username"
  password: "db pass"
  database: "db name"
  port: 3306
  1. Replace the values in the config.yml file with those of your database.
  2. Make sure the map you are running has a map config inside the schema, go into the schema gamemode folder, then check schema/config/maps/MAP NAME HERE.lua.
  3. Set the of your server to the folder name of the schema.
  4. That's it. Join the server to confirm if everything is working. If it's not working, check the console for errors and read the message on the in-game error screen.
  5. *You probably will want to edit some of the schema config. In your schema folder, navigate to config/sh_config.lua to configure your server.*
